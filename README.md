# oh-my-claudelin

Personal modular overlay on [oh-my-claudecode](https://github.com/Yeachan-Heo/oh-my-claudecode) (OMC). Adds composed skills, single-source-of-truth tool routing, and upstream-safe customization — without forking OMC's source tree.

Built on the [skill-x](https://github.com/adelin-b/skill-cross-integration) compose/sync/audit contract so derived skills auto-rebuild when their parents drift.

---

## Why an overlay, not a fork-with-patches?

Editing OMC's tree directly turns every upstream pull into a merge conflict. The overlay pattern keeps two repos:

| Repo | Role |
|---|---|
| [`adelin-b/oh-my-claudecode`](https://github.com/adelin-b/oh-my-claudecode) | Plain fork. `main` mirrors upstream byte-for-byte. Used only when a patch is worth PRing upstream. |
| `adelin-b/oh-my-claudelin` (this repo) | Additive overlay. Composed skills, decision-layer rules, scripts, personal config. Never edits OMC files. |

`git pull upstream main` on the fork is always clean. Local customization lives here and is versioned independently.

## What's in the overlay

| Path | Purpose |
|---|---|
| [`FEATURE-MAP.md`](./FEATURE-MAP.md) | **The truth source.** Every tool installed (skills, MCPs, plugins, CLIs) mapped by domain, with overlap, single-truth designation, and decision rule when multiple options coexist. |
| `skills/*-best/SKILL.md` | Composed skills derived from N parents under a `compose_rule`. Frontmatter records `composed_from`, `compose_variants`, `selected`, `selection_rationale`. |
| `compose-rules/*.yaml` | Inputs to `skill-x` compose tool. Editing a rule + recompose regenerates the SKILL.md. |
| `sync.config.json` | Parent skills tracked for drift (`skill-sync`). Triggers cascade recompose when a parent moves. |
| `scripts/` | `install.sh`, `update.sh`, `compose.sh`, `doctor.sh`. |
| `.claude-plugin/` | Claude plugin manifest + marketplace metadata. |
| `docs/ARCHITECTURE.md` | Two-repo model + overlay lifecycle. |
| `docs/ADD-A-TOOL.md` | Workflow to add a new tool: skill-x audit → map → compose-rule → SKILL.md. |

## First wave of composed skills

| Skill | Composed from | Decision rule (one-liner) |
|---|---|---|
| `browser-best` | claude-in-chrome MCP + proofshot + agent-browser | Auth/session → claude-in-chrome. Visual PR verification → proofshot. Token-efficient interaction → agent-browser. |
| `memory-best` | auto-memory + ctx-mode + omc:wiki + mex | Cross-session facts → auto-memory. Session sandbox → ctx-mode. Project knowledge → omc:wiki. Project scaffold + drift → mex. |
| `review-best` | omc:code-reviewer + omc:verifier + built-in `/review` | Local pre-commit → omc:code-reviewer. PR on GH → built-in `/review`. Completion gate → omc:verifier. |
| `docs-best` | plugin:context7 + opensrc + omc:document-specialist | Library API/version → context7. Read actual installed source → opensrc. Repo-internal docs → omc:document-specialist. |

Each composed skill carries the full skill-x frontmatter so future audits surface the rationale, not just the winner.

## Install

```bash
git clone https://github.com/adelin-b/oh-my-claudelin ~/Developer/oh-my-claudelin
cd ~/Developer/oh-my-claudelin
bash scripts/install.sh
```

Plugin path (preferred for end users):

```
/plugin marketplace add adelin-b/oh-my-claudelin
/plugin install oh-my-claudelin@adelin-b
```

## Update (daily)

```bash
bash scripts/update.sh
```

Pulls the OMC fork from upstream, runs `skill-sync` against tracked parents, runs `skill-compose` so any drifted composed skill is regenerated, re-links symlinks.

## Add a new tool

See [`docs/ADD-A-TOOL.md`](./docs/ADD-A-TOOL.md). Short version:

1. Append the tool to `FEATURE-MAP.md` under the right domain row.
2. Run `npm run audit` in the overlay (delegates to skill-x audit) to surface overlaps with existing tools.
3. If it overlaps, edit the affected `compose-rules/*.yaml` and add the tool to `composed_from`. Add a variant + rationale.
4. `bash scripts/compose.sh <domain>` to regenerate the SKILL.md.
5. Commit the map change + new compose rule + regenerated SKILL.md together.
